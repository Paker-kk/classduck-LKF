# 任务49：课程删除联动待办与关联课程去重色基线

## 1. 任务背景
1. 用户反馈课程删除后，关联待办仍有残留，要求改为彻底物理删除。
2. 用户新增同名课程颜色与待办关联课程选择器规则：
   - 同名课程默认颜色需复用基准主题色；
   - 关联课程选择器需按同名去重，并使用基准主题色；
   - 默认展示 5 条，超出走“更多”弹窗并支持搜索。
3. 要求仅改本次范围，不破坏现有功能，并完成安卓模拟器联调。

## 2. 本次改动

### 2.1 课程删除联动待办物理删除
- 文件：`classduck_app/lib/features/todo/data/todo_repository.dart`
1. 新增 `deleteTodosByCourseName(courseName)`，按课程名直接删除 `todo_item` 表记录（含未完成和已完成）。
2. 删除匹配增强为“精确匹配 + 去首尾空格匹配”，兼容历史脏数据导致的课程名空格偏差。

- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 在课程详情“删除课程”确认后，先执行 `deleteTodosByCourseName(course.name)`，再删除课程本体。
2. 保持原有删除确认交互与课程刷新逻辑。

### 2.2 同名课程基准色规则（手动添加场景）
- 文件：`classduck_app/lib/features/schedule/data/schedule_repository.dart`
1. 新增全量课程查询与手动课程查询能力：
   - `getAllCourses()`
   - `getAllManualCourses()`
2. 新增精确课程名基准色查询：`getManualCourseBaseColor(exactCourseName)`。
3. 基准色计算规则：按创建时间升序（同时间按 id 升序）取“最早创建且仍存在”的手动课程颜色。

- 文件：`classduck_app/lib/features/schedule/ui/manual_add_course_page.dart`
1. 手动添加页面监听课程名输入。
2. 当用户未手动改色时，自动按“完全同名”查询并回填基准主题色。
3. 用户一旦手动选色，后续不再被自动回填覆盖（仅作用当前条目）。

### 2.3 待办关联课程去重与主题色基准
- 文件：`classduck_app/lib/features/todo/ui/todo_page.dart`
1. 关联课程数据源改为“全量手动课程（import_type=0）”。
2. 按课程名精确去重并聚合：
   - 基准色：同名课程中最早创建课程的颜色；
   - 排序时间：同名课程中最新创建时间（用于倒序展示）。
3. 新建待办入口传递结构化课程选项（名称、主题色、排序时间）。

- 文件：`classduck_app/lib/features/todo/ui/new_todo_page.dart`
1. 关联课程区域默认展示最多 5 条；超过 5 条时显示“更多”按钮。
2. 点击“更多”弹出白底黑字课程选择弹窗，完整显示去重后的课程列表并支持搜索筛选。
3. 弹窗中点击课程项即选中并关闭；底部操作按钮采用黄底白字。

### 2.4 UI 统一
- 课程相关二级弹窗（关联课程“更多”弹窗）统一白底黑字。
- 弹窗操作按钮采用黄底白字，和现有时间设置主按钮风格统一。

## 3. 覆盖的边界场景
1. 同名课程多次添加：默认颜色自动复用基准色。
2. 手动改色：仅影响当前课程，不影响后续同名课程默认色来源。
3. 基准课程删除：基准色会随“剩余最早创建课程”自动顺延（动态计算）。
4. 同名课程全删后重建：基准色记录自动清空，后续按首次添加重新建立。
5. 关联课程少于/大于 5 条：正确切换“直接展示/更多弹窗”。
6. 删除课程后待办残留：已改为彻底物理删除，不区分已完成/未完成。
7. 跨页面回看一致性：待办页增加轻量刷新门控，避免从课表页返回后短时看到旧缓存列表。

## 4. 验证
1. 目标改动文件已通过静态错误检查，无语法/分析错误。
2. 计划在安卓模拟器回归以下路径：
   - 课程主页面删除课程 -> 待办页残留校验
   - 手动添加同名课程 -> 默认色回填校验
   - 新建待办关联课程 -> 去重、前 5 展示、更多搜索与选中校验
