# 任务61：黄色容器可见性修复与磨砂再减弱

1. 用户复测反馈两项未通过：
   - 右侧提示小容器未实际可见。
   - 覆盖已有课程区域仍偏重，底层信息可读性不足。
2. 约束：仅允许最小改动，禁止触碰已达标交互链路。

## 2. 本次改动（仅两项）

### 2.1 右侧提示容器可见性修复
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 将提示容器从 `ClipRRect` 内部移到外层 `Stack`，避免被圆角裁剪导致不可见。
2. 保留 `IgnorePointer`，继续作为纯视觉提示，不参与点击。
3. 增加边界自适应：
   - 默认显示在黄色容器右侧。
   - 当右侧空间不足时自动切换到左侧，保证可见。

### 2.2 覆盖区域磨砂再减弱
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 覆盖已有课程单元格参数下调为：
   - 透明度 `alpha = 0.10`
   - 模糊 `sigmaX/sigmaY = 3.0`
2. 未覆盖空白单元格继续保持纯品牌黄，不改逻辑。

## 3. 已确认不变项
1. 外部点击仅关闭容器。
2. 黄色容器整块可点击进入手动添加。
3. 拖拽扩选与点击跳转隔离。
4. 提示显示/隐藏时机（弹出显示、滑动隐藏、超时隐藏、关闭同步隐藏）不变。

## 4. 验证建议
1. 点击空白格后，先观察提示小容器是否稳定可见（右侧或边界回退到左侧）。
2. 在覆盖已有课程场景下拖拽，观察课程名/教师/地点是否比上一版更清晰。
3. 回归已达标项，确认无行为回退。
