# 任务55：学期已结束与周起始日列映射及冲突弹窗修复

## 1. 任务背景
1. 用户复测后反馈：
   - 需要新增“学期已结束”状态与灰态规则；
   - 每周起始日变更后，课程列未跟随重排；
   - 侧栏冲突红字未出现，且数字输入仍可触发红屏路径。
2. 要求继续保持模拟器联调与日志监测。

## 2. 本次改动

### 2.1 学期已结束状态
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 新增 `_isSemesterEnded(config)` 判断。
2. 顶部周状态文案新增“已结束”。
3. 主网格灰态显示规则调整为：未开学或已结束时可灰显；开学后非当前周课程不在主网格显示。

### 2.2 每周起始日列映射生效
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 新增 `_weekdayForColumn(dayColumn, weekStartDay)`。
2. 课程渲染构建改为按“列 -> 实际 weekday”映射筛课。
3. `_RenderedCourseBlock` 新增 `dayColumn`，课程卡横向位置改用 `dayColumn`，确保周起始日变更后课程列同步移动。

### 2.3 侧栏冲突提示与输入弹窗校验
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 上午/下午节数卡片支持卡片内红字提示，冲突态禁用 `+`。
2. 手动输入数字时，冲突态拦截“继续增加”，避免越界到冲突区。
3. 时间设置弹窗新增 `validator` 回调：
   - 冲突时在弹窗内直接红字提示；
   - 阻止确认提交，避免进入后续异常链路。

### 2.4 多课表待办隔离链路补齐
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 课程详情联动待办重命名/删除时限定当前 `tableId`。
2. 课表加载与切换时同步激活课表 ID 到仓储级通知。

## 3. 验证要点
1. 开学前显示“未开学”，学期结束后显示“已结束”。
2. 修改每周起始日后，课程列位置正确重排。
3. 侧栏冲突时卡片内红字可见，`+` 按钮禁用。
4. 时间弹窗中输入会触发冲突红字，不再因确认导致红屏。
5. 删除/查看待办时不串课表数据。
