# 任务67：课程卡长按拖动调整位置与周数重叠冲突判定

## 1. 任务目标
1. 在课表主页面新增「已有课程长按拖动调整位置」能力。
2. 冲突判定收敛为唯一标准：周数范围是否有重叠。
3. 保持现有点击详情、编辑、删除链路不被破坏。

## 2. 本次实现

### 2.1 长按拖动主链路
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 课程卡由点击组件升级为 `LongPressDraggable`，长按延迟设为 300ms。
2. 触发拖动时执行一次短震动（50ms）。
3. 拖动反馈卡片应用缩放/透明度/阴影（0.95、0.9、黑色轻阴影）。
4. 原位置显示品牌黄 10% 占位块。

### 2.2 目标磁吸与高亮
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 网格新增拖放层（`DragTarget`），按单元格吸附落点。
2. 当前落点使用 `AnimatedContainer` 显示 1dp 品牌黄边框高亮。

### 2.3 冲突规则（周数重叠）
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 仅在以下条件同时满足时判定冲突：
   - 同一星期；
   - 节次区间有交集；
   - 周数区间有交集（`dragStart <= existEnd && dragEnd >= existStart`）。
2. 周数不重叠时，即使同一格也允许落位。
3. 冲突时禁止落位并回弹，弹出提示：
   - `目标位置已有课程与该课程周数重叠，无法移动`

### 2.4 落位持久化
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
- 文件：`classduck_app/lib/features/schedule/data/schedule_repository.dart`（复用既有更新接口）
1. 落位成功后更新课程：星期、起始节次、节次数量。
2. 写库后立即刷新课表，确保切换页面和重启后位置保持。

### 2.5 手势冲突收口
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 课程拖动激活时锁定网格滚动（`NeverScrollableScrollPhysics`），避免拖动时页面跟滚。
2. 指针移动增加原生兜底（鼠标左键按住移动可触发），提升模拟器稳定性。

## 3. 已保持不变
1. 课程卡短按仍打开详情弹窗。
2. 课程详情内编辑/删除链路不变。
3. 黄色快速添加容器与其提示逻辑不受本任务影响。

## 4. 验收清单
1. 长按课程卡可进入拖动并出现激活反馈。
2. 拖动到空位可落位并刷新。
3. 拖动到同格课程：
   - 周数不重叠可落位；
   - 周数重叠被拦截并回弹。
4. 拖动期间页面不再上下跟随滚动。
5. 拖出网格后回弹，数据不变化。
