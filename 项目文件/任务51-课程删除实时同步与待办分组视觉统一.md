# 任务51：课程删除实时同步与待办分组视觉统一

## 1. 任务背景
1. 用户反馈：删除课程后立即进入待办页仍可见旧记录，需手动刷新或重启后才消失。
2. 用户要求：修复为“无感实时同步删除”，并统一“添加待办”按钮视觉到手动添加课程按钮规范。
3. 同时要求优化待办主页面折叠分组的拥挤观感，保持黄色主色、白底黑字、圆角体系。

## 2. 本次改动

### 2.1 实时同步删除闭环
- 文件：`classduck_app/lib/features/todo/data/todo_repository.dart`
1. 新增全局数据版本通知：`TodoRepository.dataVersion`（`ValueNotifier<int>`）。
2. 所有待办数据写操作（新增、改名、完成态切换、删除单条、按类型删除、按课程删除）统一触发版本通知。
3. 课程名联动删除维持“规范化匹配 + 物理删除”策略，并返回删除数量用于验收提示。

- 文件：`classduck_app/lib/features/todo/ui/todo_page.dart`
1. 页面状态新增两类监听：
   - 监听 `TodoRepository.dataVersion`，数据变化后立即 `_load()`；
   - 监听生命周期 `AppLifecycleState.resumed`，前台恢复时强制 `_load()`。
2. 移除旧的“3 秒门控自动刷新”逻辑，避免出现刚删除后短时渲染旧缓存的窗口。

- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 保持“先删关联待办，再删课程，再刷新 UI”的顺序。
2. 删除成功提示继续回显联动删除数量，便于现场复核。

### 2.2 待办创建按钮视觉统一
- 文件：`classduck_app/lib/features/todo/ui/new_todo_page.dart`
1. `添加待办` 按钮背景统一为 `AppTokens.duckYellow`。
2. 按钮文字统一为白色、粗体，与“手动添加课程”按钮保持同体系视觉。

### 2.3 待办分组折叠态 UI/UX 优化
- 文件：`classduck_app/lib/features/todo/ui/todo_page.dart`
1. 每个待办类型分组增加独立白色圆角容器与浅边框，增强分组边界识别。
2. 调整分组间距和列表内边距，提升页面呼吸感，减少黏连拥挤感。
3. 展开项改为“顶部间距”节奏，折叠与展开过渡更清晰。

## 3. 规则对照（本次验收重点）
1. 删除课程后，无需手动刷新待办页，关联待办应实时消失。
2. 待办页面前后台切换后，展示数据必须与数据库一致。
3. `添加待办` 按钮需与“保存课程”按钮在色彩和字体上保持同风格。
4. 折叠状态下所有类型分组边界清晰，视觉不拥挤。

## 4. 建议复测路径
1. 新建课程 A，并创建 2 条关联待办（未完成/已完成各 1）。
2. 在课程页删除课程 A，立刻切到待办页，验证两分栏均无残留。
3. 进入新建待办页，检查 `添加待办` 按钮是否黄底白字并与保存课程同风格。
4. 在待办主页面观察分组折叠态，确认分组间距和边界清晰。
