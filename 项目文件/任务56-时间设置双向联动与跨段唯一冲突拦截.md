# 任务56：时间设置双向联动与跨段唯一冲突拦截

## 1. 任务背景
1. 用户明确红线：
   - 同时段内部修改必须“无提示、无拦截、无回退”，保留用户修改并自动联动其他节次。
   - 跨时段边界冲突是唯一允许提示与拦截的场景，且必须是双向边界（前段末节/后段首节）。
2. 用户要求同步完成：
   - 节数输入弹窗超限回退与弹窗内红字提示。
   - 节数输入弹窗输入框视觉改为品牌黄（光标/下划线）。

## 2. 本次改动

### 2.1 同时段时间编辑改为双向联动
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 新增段内联动方法，编辑第 N 节后同时执行：
   - 向后顺推：第 N+1 及之后课程按“全局课时+课间”重算。
   - 向前逆推：第 N-1 及之前课程按“全局课时+课间”反向重算。
2. 编辑提交前先在配置草稿上演算，再原子写回，避免中途脏状态。
3. 同时段内部不再触发额外拦截提示。

### 2.2 跨时段边界冲突收敛为唯一拦截
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 统一冲突文案：
   - `您的操作会导致上午课程与下午课程时间冲突，无法完成该操作`
   - `您的操作会导致下午课程与晚上课程时间冲突，无法完成该操作`
2. 校验改为双向边界：
   - 上午↔下午、下午↔晚上均支持“前段末节”与“后段首节”触发。
3. 冲突时阻止提交，保留原合规配置。

### 2.3 节数调整规则与弹窗增强
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 上午/下午节数 `+` 在达到合规上限后直接置灰禁用。
2. 节数输入弹窗新增上限回退：输入超出边界时自动回退到最大合规值，并在输入框下显示红字提示。
3. 侧栏时段卡片支持对应冲突红字，提示与操作位置一致。

### 2.4 输入框视觉统一为品牌黄
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 节数输入弹窗数字框：光标、下划线、聚焦下划线统一为 `AppTokens.duckYellow`。
2. 时间编辑弹窗（小时/分钟输入）同步统一为品牌黄，保持白底黑字与黄底白字确认按钮。

## 3. 验证要点
1. 同时段内：
   - 修改靠前节次会向后顺推。
   - 修改靠后节次会向前逆推。
   - 无跨段冲突时不出现任何提示。
2. 跨时段边界：
   - 前段末节越界、后段首节越界都能被拦截。
   - 仅出现两条约定冲突文案之一。
3. 节数输入：
   - `+` 在边界上限时为灰色禁用。
   - 输入超限会回退到最大合规值，并在弹窗内出现红字提示。
4. 运行稳定：
   - 不出现时间设置相关红屏；保存后课表渲染正常。
