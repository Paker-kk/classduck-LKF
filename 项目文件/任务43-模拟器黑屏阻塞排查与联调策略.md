# 任务43：模拟器黑屏阻塞排查与联调策略

## 1. 任务背景
1. 用户反馈 Android 模拟器处于黑屏，无法继续页面验收。
2. 目标是先恢复可观察运行态，再继续导入链路验收。

## 2. 问题现象
1. `flutter run -d emulator-5554 --debug` 可编译安装，但随后出现 `Lost connection to device`。
2. 启动日志持续出现渲染相关异常信号：
- `Width is zero. 0,0`
- `OpenGLRenderer` swap behavior 匹配失败
3. 调试链路断开后，模拟器仅显示黑屏界面，无法稳定进入业务页。

## 3. 已执行处理
1. 清理并终止了残留 `flutter run` 终端会话，避免批处理/会话冲突。
2. 尝试了以下启动方式：
- `--debug`（仍会断连）
- `--enable-software-rendering --no-enable-impeller`（仍有服务协议握手失败）
- `--release`（成功完成安装并维持运行态）
3. `--release` 首次安装时发生签名不一致：
- `INSTALL_FAILED_UPDATE_INCOMPATIBLE`
- 已自动卸载旧包并重新安装成功。

## 4. 当前结论
1. 主要阻塞来自模拟器渲染/调试通道稳定性，而非业务功能逻辑报错。
2. 当前联调建议暂以 `release` 运行作为验收通道，待用户确认可见页面后继续功能验收。

## 5. 下一步计划（待用户意见）
1. 若用户确认已亮屏：继续执行“导入链路”验收。
2. 若仍黑屏：切换 AVD 图形配置（软件渲染/冷启动）后再验证。
3. 完成验收后，再统一收口临时联调策略说明。
