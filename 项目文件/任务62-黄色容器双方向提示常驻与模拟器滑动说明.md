# 任务62：黄色容器双方向提示常驻与模拟器滑动说明

## 1. 任务背景
1. 用户反馈：右侧提示已可见，但仍缺少左右方向提示。
2. 用户反馈：拖拽后提示会消失，不利于持续引导。
3. 用户提问：安卓模拟器中如何稳定执行上下/左右拖拽。

## 2. 本次改动

### 2.1 双方向提示补齐
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 保留侧边提示容器（上下箭头）。
2. 新增横向提示容器（左右箭头）：
   - 默认显示在黄色容器上方；
   - 若上方空间不足，自动显示在黄色容器下方。

### 2.2 提示改为常驻显示
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 移除“滑动后隐藏/3秒自动隐藏”策略。
2. 规则改为：只要黄色容器存在，提示一直显示；关闭黄色容器时提示同步消失。
3. 保持纯视觉属性：两个提示容器均为 `IgnorePointer`，不拦截点击。

### 2.3 保持不变项
1. 外部点击仅关闭容器。
2. 容器整块可点击跳转手动添加。
3. 拖拽扩选与点击跳转逻辑不变。
4. 覆盖态磨砂继续沿用上一版轻量参数（10% + sigma 3.0）。

## 3. 安卓模拟器滑动操作说明
1. 先点击课表空白格，出现黄色容器。
2. 上下拖拽：
   - 鼠标左键按住黄色容器不放，向上或向下移动鼠标，再松开。
3. 左右拖拽：
   - 同样按住左键不放，向左或向右移动鼠标，再松开。
4. 关键点：
   - 必须是“按住左键拖动”，不是滚轮。
   - 若触控板可用，可用“按住并拖动”手势，逻辑同鼠标左键。

## 4. 验收建议
1. 观察是否同时存在“上下提示 + 左右提示”。
2. 连续拖拽后提示是否仍保持显示，直到关闭容器才消失。
3. 验证上边界场景：容器靠近顶部时，左右提示应自动切换到容器下方。
