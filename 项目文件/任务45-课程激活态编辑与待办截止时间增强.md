# 任务45：课程激活态编辑与待办截止时间增强

## 1. 任务背景
1. 用户要求课程主页面二级详情支持可编辑与保存，覆盖课程名、星期、周次、节次、教师、地点。
2. 用户要求保留既有“左右滑动查看同时间段不同课程”能力，不得破坏。
3. 用户要求课程详情待办区补充截止时间，并按“明天/后天/其他日期”智能显示。
4. 用户确认保存策略：关闭弹窗时统一保存，但编辑态内容必须即时可见。

## 2. 本次改动

### 2.1 课程详情可编辑化（统一保存）
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 课程激活弹窗升级为可编辑表单结构：
- 课程名称（文本输入）
- 星期（下拉选择）
- 周次（起止数字输入）
- 节次（起止数字输入）
- 教师（文本输入）
- 地点（文本输入）
2. 保持编辑态即时可见：输入后弹窗内立即反映最新内容。
3. 关闭弹窗触发统一保存：
- 先执行全量校验
- 校验通过后批量提交课程修改
- 校验失败则提示并阻断关闭

### 2.2 表单校验机制
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 课程名不能为空。
2. 星期范围限制为 1-7。
3. 周次要求数字且满足 `1 <= start <= end <= 30`。
4. 节次要求数字且满足 `1 <= start <= end <= maxPeriod`。

### 2.3 待办区截止时间增强
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 待办列表项改为左右信息结构：左侧任务名，右侧截止时间。
2. 截止时间显示规则：
- 明天：显示“明天”
- 后天：显示“后天”
- 其他日期：显示 `MM-dd`

### 2.4 数据层与关联同步
- 文件：`classduck_app/lib/features/schedule/data/schedule_repository.dart`
1. 新增 `updateCourseDetail`，用于更新课程核心字段并写入 `updated_at`。

- 文件：`classduck_app/lib/features/todo/data/todo_repository.dart`
1. 新增 `renameCourseName`，用于课程改名后同步更新关联待办的 `course_name`。

- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 在课程弹窗保存回调中串联：课程更新 -> 待办课程名同步 -> 主课表刷新。

### 2.5 交互约束（保持原能力）
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
1. 保留原有横向滑动切课手势与左右箭头切换逻辑，不改动行为。
2. 为保证“关闭即保存”流程稳定，课程激活弹窗改为不可点击遮罩关闭，仅允许通过弹窗内关闭按钮触发保存退出。

- 文件：`classduck_app/lib/shared/widgets/duck_modal.dart`
1. `DuckModal.show` 新增 `barrierDismissible` 参数（默认 `true`），支持对关键弹窗按需关闭外部点击退出。

## 3. 验证项
1. 静态错误检查：新增与改动文件均无语法/分析错误。
2. 功能联调：待 Android 模拟器实机联调确认交互细节与视觉对齐。

## 4. 风险与后续
1. 当前待办与课程关联键仍是课程名，若后续需要跨同名课程精确关联，建议升级为 `courseId` 级关联。
2. 若后续产品希望恢复“点击遮罩关闭”，需在弹窗增加“放弃修改/保存修改”分支确认。
